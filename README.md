### 代码提交规范

```
格式 : <type>(<scope>): <subject>
// type 必填表示提交类型
// scope 选填表示 commit 的作用范围，如数据层、视图层，也可以是目录名称
// subject 必填用于对 commit 进行简短的描述

例如:
  feat - 新功能 feature
  fix - 修复 bug
  merge - 合并分支
  docs - 文档注释
  style - 代码格式(不影响代码运行的变动)
  refactor - 重构、优化(既不增加新功能，也不是修复 bug)
  perf - 性能优化
  test - 增加测试
  chore - 构建过程或辅助工具的变动
  revert - 回退
  build - 打包
  例：git commit -m 'chore:新建基础构建'
```
